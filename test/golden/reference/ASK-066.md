# ASK-066 参考答案

## 看大小分布

```bash
du -sh .codeclaw/backups/*/ | sort -rh | head -20
# 或
ls -lt .codeclaw/backups/ | head -20  # 按修改时间倒序
```

batch dir 命名是 `yyyymmdd-HHMMSS-rand4`，可直接看日期。

## 安全清理（保留 7 天）

```bash
# Dry run：先看哪些会被删
find .codeclaw/backups/ -maxdepth 1 -type d -mtime +7 -print

# 确认 OK 再删
find .codeclaw/backups/ -maxdepth 1 -type d -mtime +7 -exec rm -rf {} +
```

要点：
- `-maxdepth 1`：只看一层 batch dir，不递归到 batch 内
- `-mtime +7`：修改时间 7 天前
- 当天的 batch 不会被命中（mtime 是今天）

## 不要这么做

```bash
# ❌ 会清掉 sessions / data.db / audit.db
rm -rf .codeclaw

# ❌ 会清掉**所有** backup（含当天）
rm -rf .codeclaw/backups
rm -rf .codeclaw/backups/*

# ❌ 没 maxdepth 1，可能进 batch 内乱删
find .codeclaw/backups -mtime +7 -delete
```

## 长期方案

在 `scripts/nightly.mjs` 加 prune 步骤：保留 30 天 batch，更老的归档或删。
