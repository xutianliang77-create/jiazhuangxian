# ASK-065 参考答案

## 配置层

| 假设 | 验证 |
|---|---|
| providers.json 缺 / current 指错 | `cat ~/.codeclaw/providers.json` 看 `current`；`codeclaw doctor` 看 provider 状态行 |
| baseUrl / apiKey 错 | 直接 `curl <baseUrl>/models -H "Authorization: Bearer $KEY"`；非 200 即认证失败 |

## 模型层

| 假设 | 验证 |
|---|---|
| LM Studio 模型未 load | `curl http://127.0.0.1:1234/v1/models \| jq` 看 list；空就是没 load |
| 选了 reasoning 模型，content 全跑到 reasoning_content | curl 一次 chat completion，看流式末尾 chunk 是 `delta.content` 还是 `delta.reasoning_content`；codeclaw 已修 fallback（commit 0e45b9d）但要确认部署是这个版本 |

## 解析层

| 假设 | 验证 |
|---|---|
| max_tokens 太小 → 输出截断到空 | reports/*.jsonl 看 answer 字段长度；commit `63a9489` 后 OpenAI 兼容路径默认 max_tokens=1024 |
| SSE 末尾 chunk 解析丢 | 开 debug log（`DEBUG=codeclaw:* npm run golden:ask -- --real --id ASK-001`） |
| 语言不匹配 | must_mention 写中文但模型回英文（或反之）；scorer 大小写归一化但语言区分 |

## 不要做

- 卸载重装 codeclaw：跟 LLM 链路无关
- 关网：本地 LLM 也不需要外网
