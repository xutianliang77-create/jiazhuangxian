# ASK-038 参考答案

## 1. 改哪个文件

`~/.codeclaw/providers.json`（首次跑用 `codeclaw setup` 引导生成；存在后直接编辑）

## 2. 字段填什么

```json
{
  "providers": [
    {
      "type": "lmstudio",
      "baseUrl": "http://127.0.0.1:1234/v1",
      "model": "qwen/qwen3.6-35b-a3b",
      "apiKeyEnvVar": "",
      "timeoutMs": 120000
    }
  ],
  "current": "lmstudio"
}
```

| 字段 | 值 | 说明 |
|---|---|---|
| `type` | `"lmstudio"` 或 `"openai"` | LM Studio 是 OpenAI 兼容协议，两个都行 |
| `baseUrl` | `http://127.0.0.1:1234/v1` | 必带 `/v1` 后缀 |
| `model` | LM Studio UI 中显示的 model id | 如 `qwen/qwen3.6-35b-a3b` |
| `apiKeyEnvVar` | `""` 或不填 | LM Studio 默认不校验 key |
| `timeoutMs` | 120000 (建议) | 本地大模型 latency 高，默认 30s 不够 |

## 3. 验证连通性

```bash
# 看 LM Studio 当前 load 哪个 model
curl http://127.0.0.1:1234/v1/models

# 或 doctor 看 provider 状态
codeclaw doctor | grep -A2 lmstudio
```

进一步真测：
```bash
curl http://127.0.0.1:1234/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model":"qwen/qwen3.6-35b-a3b","messages":[{"role":"user","content":"hi"}],"max_tokens":10}'
```
返回 `"choices":[{"message":{"content":"..."}}]` 即通。
