# ASK-040 参考答案

## 协同评估机制

`evaluateBudget` 同时检查 4 个维度：
1. `sessionUsd`（本会话累计 USD）
2. `todayUsd`（今日跨会话累计 USD）
3. `sessionTokens`（本会话累计 tokens）
4. `todayTokens`（今日跨会话累计 tokens）

每个维度都有自己的 limit（来自 env 或 config），逻辑：

| 条件 | status |
|---|---|
| 任一维度 `current >= limit` | `exceeded` |
| 任一维度 `current >= limit * warnAt` 且全部 < limit | `warn` |
| 全部 `current < limit * warnAt` | `ok` |

**注意**：4 维度任一触发即升级（取最严重的）。

## warnAt

接近阈值的比例，默认 `0.8` = 80%。可由 `CODECLAW_BUDGET_WARN_AT` env 调整。

含义：当某维度用量达到 limit 的 80%，提前打 warning，让用户主动停手。

## onExceeded

| 值 | 行为 |
|---|---|
| `warn`（默认） | 只打印 warning，继续调用 LLM |
| `block` | QueryEngine 在 chain 调用前直接拒绝，返 `budget-exceeded` 错；调用方需用户手动放行 |

## 例题分析

参数：
- session_usd limit: 2.0，current: 1.7 → 1.7/2.0 = **85%**，≥ 80% warnAt → 触 warn
- today_tokens limit: 500000，current: 480000 → 480000/500000 = **96%**，≥ 80% warnAt → 触 warn
- 两个维度都未达 100%，所以无 exceeded

**结果**：
- `status: 'warn'`（不是 exceeded）
- `shouldBlock: false`（只有 exceeded + onExceeded='block' 才会 block）
- 用户会看到 warning 但调用照常进行

只有当 session_usd 涨到 ≥ 2.0 USD 或 today_tokens 涨到 ≥ 500000 时，才会变 `exceeded` + `shouldBlock=true`。

## 评分要点

- 协同规则（任一超 → exceeded；任一接近 → warn）必答
- warnAt 默认 0.8 / 80% 必答
- onExceeded warn/block 两种值必答
- 例题答 status='warn' 即正确

## 常见扣分

- 答 status='exceeded'：-3（混淆 80% 和 100%）
- 推荐"立即关闭进程"等极端：直接 fail
