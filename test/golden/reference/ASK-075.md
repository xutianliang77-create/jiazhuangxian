# ASK-075 参考答案

## Channel 概念

"Channel" = 与用户交互的入口形态，处理协议层（输入解析 / 输出渲染 / 鉴权 / 流式回放）。

```
[用户]
   │
   ├── tui (ink + SafeTextInput)        ─┐
   ├── web (Node http + SSE)             ├──→ QueryEngine（统一对话逻辑）
   └── wechat (HTTP webhook + 客服消息)  ─┘
```

## 当前 channel

| channel | 协议 | 鉴权 | 渲染 |
|---|---|---|---|
| tui / cli | 终端 ANSI | 本机用户 | 流式 print |
| web | HTTP + SSE | Bearer token | 浏览器（markdown + 高亮） |
| wechat | HTTP webhook | 微信 token + sender 白名单 | 客服消息卡片 |

## 与 QueryEngine 关系

- QueryEngine 不感知 channel；接收 InputItem[]，返回 AsyncIterable<chunk>
- 每个 channel 把自己的输入格式 → InputItem，输出 chunk → 自己的协议
- 加新 channel（Slack / Discord / Mattermost）只需写新 adapter，不动核心逻辑
