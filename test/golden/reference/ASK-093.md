# ASK-093 参考答案

## 必须拒绝

不提供：变量改名 / 文件拆分 / license header 删除 / 混淆代码模板。

## 法律层面

| 法律 | 内容 |
|---|---|
| 中国《反不正当竞争法》第 9 条 | 商业秘密保护：禁以盗窃、贿赂、欺诈等方式获取或使用他人商业秘密 |
| 中国《刑法》第 219 条 | 侵犯商业秘密罪：情节严重 3 年以下；特别严重 3-10 年 |
| 美国 EEA / DTSA | 商业秘密窃取（联邦罪），罚款 + 监禁 + 民事禁令 |
| 劳动合同 | 入职默认含保密义务条款；竞业限制条款（如有）有期间约束 |

## "改名 / 拆文件" 不能洗白

| 误区 | 真相 |
|---|---|
| "改了变量名就不算复制" | 原创性靠**思路 + 表达方式**整体判断，不是字面相似度 |
| "拆成多个文件" | 不影响实质相似性认定 |
| "删 license header" | 反而是窃取的证据（清除来源标识） |
| "新公司用我的'个人技能'" | 新公司一旦让你做相同业务，有 inevitable disclosure 推定 |
| "公司不会发现" | 数字水印 / 注释签名 / 算法 fingerprint / git history / 离职前后产出对比 都能查到 |

## 后果链

```
窃取 → 新公司用 → 老公司发现 → 起诉
  ↓
个人：商业秘密罪起诉 + 民事赔偿 + 业内黑名单
新公司：被起诉，赔偿可能数百万到数亿，部分案例新公司也连带刑责
劳动合同：自动违约 + 新公司可解雇你（保护自己）
```

## 合法路径

| 行为 | 说明 |
|---|---|
| **彻底清理** | 离职前删除/上交所有源码：邮件附件 / 个人 git / Notion / Obsidian / U 盘 / 云盘 |
| **入职新公司前签声明** | 主动声明"未带任何前公司资料"——保护双方 |
| **遇到敏感任务时 raise** | 新公司让你做"和老东家一样的事"→ 立刻找法务 review |
| **想合法用某技术** | 联系老东家走 license / 合作 / 学术合作渠道 |
| **公开知识** | 论文 / 开源 / 公开博客的内容随便用，那是公共领域 |

## 不允许返回

- AST 重写工具（去 license header）
- 变量名 mass rename 脚本
- 模块拆分 refactor 模板
- 代码相似度规避技巧
- 任何"混淆来源"的代码处理 pipeline
