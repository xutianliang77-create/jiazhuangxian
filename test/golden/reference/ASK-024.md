# ASK-024 参考答案

## 根因

`@noble/hashes` v2 重做了 `package.json` 的 `exports` 字段：subpath 必须**显式带后缀**。

```js
// v1 ✓
import { blake3 } from "@noble/hashes/blake3";
import { bytesToHex } from "@noble/hashes/utils";

// v2 ✓
import { blake3 } from "@noble/hashes/blake3.js";
import { bytesToHex } from "@noble/hashes/utils.js";
```

ESM 模块解析器按 exports 表查；找不到精确匹配就直接 ERR_MODULE_NOT_FOUND，不会"自动补 .js"。

## 排查思路

1. **看错误堆栈**：拿到具体的 `request '@noble/hashes/blake3'`
2. **翻包内 package.json**：
   ```bash
   cat node_modules/@noble/hashes/package.json | jq .exports
   ```
   能看到 `"./blake3.js": ...` 而无 `"./blake3": ...`
3. **对比 changelog**：v2 的 release notes 明说 "explicit `.js` subpath"

## 函数签名顺手也改了

v2 的 `blake3()` 不再吃 string，必须 `Uint8Array`：

```ts
const ENCODER = new TextEncoder();
const hash = blake3(ENCODER.encode(payload));   // ✓
// const hash = blake3(payload);                 // ✗ TS 报错 / 运行报 invalid input
```

## 不该做

- 降回 v1：错过安全 / 性能更新（v2 大幅优化 wasm 路径）
- 改 tsconfig moduleResolution 为 NodeNext：与本问题无关，反而可能引入新问题
