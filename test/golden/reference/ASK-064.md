# ASK-064 参考答案

## 为什么强制

Web channel 默认监听 `127.0.0.1:7180`，但 loopback 不等于安全：
- 同机其他进程 / 容器 / 浏览器扩展 都能 fetch 它
- 若被反向代理/隧道暴露公网，没 token = 任何人调

所以 `startWebServer` 启动时检查 `CODECLAW_WEB_TOKEN`，缺则拒绝启动。

## 生成 token

```bash
# 推荐
openssl rand -hex 32

# 或
node -e "console.log(require('crypto').randomUUID())"

# 或
uuidgen
```

## 设置

```bash
# 临时
export CODECLAW_WEB_TOKEN="<rand>"
codeclaw web

# 持久（.env，必须在 .gitignore）
echo "CODECLAW_WEB_TOKEN=$(openssl rand -hex 32)" >> ~/.codeclaw/.env
```

## 客户端调用

```bash
curl -H "Authorization: Bearer <TOKEN>" http://127.0.0.1:7180/v1/web/sessions
```

## Rotate

1. 重新生成 token
2. 替换 env / .env 值
3. 重启 web 进程（旧 token 立即失效）
4. 通知用过老 token 的客户端更新

## 不该做

- 把 token 检查改成可选 / 注释掉 → 直接破坏鉴权
- 把 token 写进 git 仓库 / Slack / 截图分享
