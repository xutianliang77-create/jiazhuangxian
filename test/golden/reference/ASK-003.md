# ASK-003 参考答案

## 答题要点

- `prev_hash` 首条事件使用"全 0 / 空串 / GENESIS"占位（实现上是 0 字符串，见 `src/storage/auditLog.ts`）
- 每条 event 的 hash = BLAKE3( prev_hash || event_id || actor || action || resource || decision || timestamp )
- 下一条事件把前一条的 hash 写进 `prev_hash` 字段，形成链

## 为什么链式而不是单条

| 攻击 | 单条 hash 是否能防 | 链式 hash 是否能防 |
|------|------|------|
| 改某一条事件内容 | 可以（hash 失配） | 可以 |
| 删除某一条事件 | ❌ | ✅（后续链断） |
| 插入伪造事件 | ❌ | ✅（后续链断） |
| 交换两条事件顺序 | ❌ | ✅ |

所以链式哈希能检测"删 / 插 / 换序"这三类攻击，不是单条哈希能做到的。

## 常见扣分 / 幻觉

- 说"类似区块链，有工作量证明" → 错；CodeClaw 只有单向哈希链，无共识
- 说"prev_hash 是加密出来的" → 错；是哈希不是加密（无法反解）
- 少提链式 / 篡改检测 → 扣分
