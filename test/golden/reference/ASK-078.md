# ASK-078 参考答案

## 三步链路

### ask

- 构造 `PermissionRequest`：tool 名 / 参数 / 风险等级 / token / 上下文摘要
- 把请求"端"出来：TUI 弹窗 / web approval card / wechat 客服消息 / 在 auto 模式下直接 short-circuit
- 不动 db

### decide

- 用户选 yes / no（人工模式）
- 或按 PermissionMode（dontAsk / auto / plan / acceptEdits / bypassPermissions）自动判定
- 计算最终 `Decision = { allowed, reason, mode }`

### record

- 写 `audit.db` 一条 event（含 prev_hash + payload_hash）
- 写 `data.db` approvals 表（业务侧 list / forget 用）
- 失败重试 / 错误处理统一在这里

## 为什么单抽 record

| 理由 | 说明 |
|---|---|
| **可测试性** | ask 步骤涉及 TUI / web stream，测试要 mock；record 是纯 db 写，单测易 |
| **多 channel 复用** | tui / web / wechat 各自的 ask 实现不同；record 共用 |
| **模式扩展点** | decide 可被 dontAsk / auto / plan 等替换；record 始终运行 |
| **审计强保证** | 把 db 写抽到独立函数，控制流不走分支；ask 异常退出 record 不被跳过 |

不是"性能优化 / 代码量"理由。
