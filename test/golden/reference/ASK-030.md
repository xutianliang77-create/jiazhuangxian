# ASK-030 参考答案

## 最佳选择

```
/bash git log -n 10 --oneline -- src/agent/queryEngine.ts
```

要看每次改动文件统计：

```
/bash git log -n 10 --stat -- src/agent/queryEngine.ts
```

要看 author 与时间：

```
/bash git log -n 10 --format='%h %ad %an %s' --date=short -- src/agent/queryEngine.ts
```

## 为什么是 /bash + git log

- git 是仓库一手数据，输出**确定性 100%**
- read-only，0 副作用，0 审批负担
- 一条命令出全部信息
- LLM / planner 不需要参与，cheap

## 为什么不用其他

| 工具 | 为什么不合适 |
|------|--------------|
| `/read` | 读文件**当前内容**，不能展开历史 |
| `/symbol` | 查符号**定义位置**，错维度 |
| `/orchestrate` | 杀鸡用牛刀，规划+执行+反思全套，对单条查询零收益 |
| GitHub 浏览器 | 跳出 REPL 上下文，回答里贴不便 |
| IDE | 同上，且 IDE 与 REPL 的协作是另一个工具问题 |

## 一句话原则

**读历史 = git log**，不要绕远路。
工具选择第一性原理：能用最简单 read-only 命令解决的事，不要拉重武器。
