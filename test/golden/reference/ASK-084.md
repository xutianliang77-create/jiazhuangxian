# ASK-084 参考答案

## 三段流水

### 先：盘点变更

```bash
git diff --stat              # 改了哪些文件
git diff                      # 看具体行
git diff --name-only HEAD~1   # 只看文件名
```
read 关键 hunk 周围的上下文，避免改完不知道动了什么。

### 中：影响范围

```bash
npm run typecheck             # 类型错误 / 引用不存在
```
对每个改动函数 / type，用 symbol find-references 或 `rg -n '\b<name>\b'` 看：
- 还有谁在用？
- 调用方是否兼容新签名？

### 后：跑测试

```bash
# 局部
npm test -- test/path/to/module.test.ts

# 全套
npm test

# 只跑 fail
npm test -- --bail
```

## 关键原则

- **typecheck 不通过就不要试 test**（tsc 错误时 vitest 也跑不了）
- **局部 test 通过 ≠ 全套通过**：跨模块影响只能跑全套发现
- **跑 pre-push gate 之前自己先跑**：避免 push 时被拦才发现

## 不要做

- `git push --no-verify`
- `git commit -m '...' [skip ci]`
- "我只改了一行应该没事"：往往就是这一行炸 cross-module
