# ASK-080 参考答案

## 模型本身被滥用（输入边界 / Prompt 攻击）

| T# | 内容 |
|---|---|
| T1 | Prompt injection：用户在自己输入里塞"忽略上述指令"伪造 system role |
| T2 | 文档投毒：从历史消息 / loader 加载的内容里夹隐藏指令 |
| T3 | Sender 假冒：多用户 channel（wechat）里恶意 sender 假冒别人 |
| T6 | 供应链污染：第三方 skill manifest 注入 control char / 隐藏 prompt |
| T7 | MCP transport 污染：远端 MCP server 返回的内容含 injection |

## 系统外围被滥用（执行边界 / 工具与文件层）

| T# | 内容 |
|---|---|
| T4 | 命令注入：bash 工具参数没 escape，shell metachar 被解释 |
| T5 | 路径穿越：read/write 工具用 `../../../etc/passwd` |
| T8 | Prompt 中夹敏感数据：向 LLM 误传 sk-/ghp_ 等密钥 |
| T11 | Token 文件权限：providers.json / .env 默认 0644 被本地读 |
| T13 | Bash 缺沙箱：personal edition 不沙箱，靠用户审批 |
| T16 | Write 冲突：多进程同时写同文件 |
| T18 | .gitignore 错配：sessions / .env 被 commit |
| T19 | Doctor 缺安全 banner：用户不知道 bash 不沙箱 |

## 区分维度

- **模型滥用** = LLM 推理过程中的"语义注入"，靠 sanitize input / sender whitelist / loader 过滤
- **外围滥用** = LLM 之外的系统行为，靠 escape / chmod / 沙箱 / .gitignore / banner

两类威胁的防御层级不同：
- 模型层威胁：守在 prompt construction 阶段（不能 100% 防，靠 sanitize + LLM 训练）
- 外围层威胁：守在工具执行 / 持久化 / 配置 / 部署阶段（可以系统化防）
