# ASK-054 参考答案

## 两条路径

### ProviderRequestError（含 status code）

| status | 分类 |
|---|---|
| 401 / 403 | auth |
| 408 / 425 / 429 | transient |
| 5xx (500-599) | transient |
| 413 / 422 | content |
| 其他或缺失 | transient（缺）/ unknown（其他） |

### 普通 Error（无 status code）

按 `message` 字段做正则匹配：

```
/ECONNRESET|ETIMEDOUT|ENOTFOUND|ECONNREFUSED|EAI_AGAIN|socket hang up|fetch failed|network/i
```

命中 → `transient`；
另有 `/aborted|AbortError/i` → `abort`；
都不命中 → `unknown`（chain 仍会尝试 retry 一次再 fallback）。

## 设计意图

- 网络抖动多以 errno 形式抛出（ECONN…）→ 标 transient 让 chain retry 同 provider
- DNS 失败（ENOTFOUND / EAI_AGAIN）短暂故障也归 transient
- `fetch failed`：Node undici 在底层网络问题时常打这个 message
