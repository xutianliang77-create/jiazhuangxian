# ASK-004 参考答案

## 答题要点

`markBodyMissing(messageId)`（`src/storage/repositories/l1MemoryRepo.ts`）是**恢复 / 回退模式**的占位：

- 元信息（`l1_memory` 表里的 message_id / seq / role / timestamp / token_cost）**还在**
- 正文（`sessions/{sid}/transcript.jsonl` 对应那一行）**丢了 / 不可读**

## 触发场景

- 磁盘损坏导致 jsonl 读取失败
- 用户手动删除 transcript.jsonl
- 跨机迁移时只搬了 data.db 没搬 sessions/
- 未来"只导出 meta"的备份方案解压回来

## 为什么这么设计

- UI 仍能按 seq 顺序占位显示（灰色 "body missing"），不至于整段对话塌
- 之后如果有备份可以补回正文，只需清 `body_missing=0`
- db 是真相来源；jsonl 是正文仓库；两者解耦，单独损坏不致命

## 常见扣分 / 幻觉

- 说"数据全丢了，不可恢复" → 错；元信息保留
- 说"用户主动删除消息" → 错；这是系统级异常标记，不是用户 UI 删除
