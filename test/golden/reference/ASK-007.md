# ASK-007 参考答案

## 答题要点

`src/commands/doctor.ts` 的 /doctor 覆盖以下分段：

| 分段 | 具体检查 |
|------|----------|
| `storage` | `~/.codeclaw/data.db` / `audit.db` 存在 & 可打开，schema_version 最新，WAL 开启 |
| `runtime` | Node 版本、平台（linux/mac/win）、cwd、可写临时目录 |
| `libs` | 关键依赖版本：better-sqlite3 / ulid / @noble/hashes / pino |
| `tokenFile` | iLink WeChat token 文件路径 / 存在性 / 权限 600 |

## 为什么要分这几段

- storage：迁移和连接问题是常见安装错误
- runtime：Node / 平台不匹配是另一大类问题
- libs：native 绑定（better-sqlite3）经常和 Node 小版本打架，先报版本
- tokenFile：W1 改版后路径从 `~/.claude/` 迁 `~/.codeclaw/`，易配错

## 常见扣分 / 幻觉

- 回答 `npm audit` / GPU / Docker 这类不存在段 → fail
- 全部说成"检查数据库" → 粒度不够，扣分
- 遗漏 tokenFile → 可接受（W1 新加），但提到是加分项
