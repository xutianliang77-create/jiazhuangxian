# ASK-002 参考答案

## 答题要点

ObservationRepo 双写策略（见 `src/storage/repositories/observationRepo.ts`）：

- 阈值：`OBSERVATION_OVERFLOW_THRESHOLD = 100 * 1024`（= 100 KB）
- `body ≤ 100 KB` → append 到 `sessions/{sid}/observations.jsonl`，`body_inline=1`
- `body > 100 KB` → 独立文件 `sessions/{sid}/observations/{stepId}.jsonl`，`body_inline=0`
- `body_path` 存相对 `sessions/{sid}/` 的路径

## 为什么这样做

- 小正文集中写合并 jsonl，减少文件数（避免小文件膨胀）
- 大正文单开文件，避免把合并 jsonl 撑得无法逐行 tail/grep
- `body_inline` 让读取端知道要"jsonl 按行找"还是"整文件 = 一条"

## 常见扣分 / 幻觉

- 阈值答成 1 MB / 10 KB / 可配置 → fail
- 答"正文也入 db" → 错误（db 只存 meta）
- 说"所有 observation 都放同一个文件" → 错误（溢出独立）
