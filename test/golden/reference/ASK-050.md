# ASK-050 参考答案

## 正则

`^(\d+)_(.+)\.sql$` —— 文件名必须以"一段数字 + 下划线 + 任意名字 + .sql"形式命名。例：

- ✅ `001_init.sql`
- ✅ `042_add_skills_table.sql`
- ❌ `init.sql`（缺数字前缀）
- ❌ `001-init.sql`（用 `-` 不是 `_`）

## 不匹配的后果

`migrate.ts` 直接 `throw new Error('bad migration filename: <file>')`，启动失败。**不会"宽松跳过"**。

## 为什么严格

- 数字段是版本号，决定执行顺序；少一个或重复都会让 schema 不一致
- 已应用的 migration 用 (kind, version) 主键记录在 `_migrations` 表；模糊匹配会让幂等保护失效
