# ASK-008 参考答案

## 用法

```
/approve-pending             # 或 /approve-pending list
/approve-pending approve <token>
/approve-pending deny <token>
```

- 空参 / `list` → 打印当前所有 pending，带 token / 请求摘要 / 创建时间
- `approve <token>` → 通过指定 token 对应的审批，原请求继续执行
- `deny <token>` → 拒绝该审批，原请求返回"用户已拒绝"

## token 在哪里来

Worker 需要审批时会把 token 写到 `~/.codeclaw/data.db` 的 `approvals` 表，并在 CLI 侧打印一个简短 token 提示。`list` 输出就来自这个表，按 ROWID 排序（见 ASK-006）。

## 关键注意

- 没有 `--all` 批量通过的用法（设计上强制逐条判断）
- 拒绝不是"忽略"，是返回显式 `denied` 给 worker，后者会把"用户已拒绝"带进审计链
- Ctrl+C 不等于批准，也不等于拒绝 —— 只是中断 CLI 本轮，审批仍 pending

## 常见扣分 / 幻觉

- 杜撰 `--all` / `--force` 参数 → fail
- 答"按回车 = 批准" → fail
- 少答 deny → 扣分
