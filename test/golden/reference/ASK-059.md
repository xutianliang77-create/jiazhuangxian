# ASK-059 参考答案

## 当前状态：占位（placeholder）

manifest schema 已经为 signature 预留：
```yaml
signature:
  algo: "ed25519"        # 算法名
  publicKey: "base64..."  # 验签公钥
  value: "base64..."      # 签名值
```

**但 loader 当前不做真正的密码学验签**。

## loader 的完整性检查

唯一的强制规则：**三字段必须同时存在**。
- 只有 `algo` 没 `value` → reject
- 有 `value` 没 `publicKey` → reject
- 全有但格式错（非 base64）→ reject

这等于"声明协议位"：将来切到真验签时，schema 不变。

## 为什么 P1 不做真验签

| 理由 | 说明 |
|---|---|
| 来源可信 | skill 只在本机 install（用户自己拿到的），不是远端 download |
| install 阶段已校验 | `validateManifest` 已检查必填字段 + 控制字符 + 长度 |
| 远端 registry 是 P2 | 等 skill marketplace 上线后再做真验签（公钥分发 / 撤销链 / TUF 等是另一个工程） |

不应说"已用 X.509 / openssl 真验签"——这是误导。
