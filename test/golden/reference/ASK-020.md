# ASK-020 参考答案

## 老 regex

```ts
needle.split(/\s*(?:或|\/|or|\|)\s*/i)
```

`or` 没加 `\b` 词边界，匹配子串即可。

## 触发的 bug

输入 `"proof-of-work"`：
- 字符位置：`p r o o f - o f - w o r k`
- 索引 10-11 是 `o` `r` → 匹配 `or`
- split 结果 `["proof-of-w", "k"]`

scorer 把每个 alternative 当 needle 在答案里 `includes`。
`"proof-of-w"` 几乎不会命中，但单字符 `"k"` 几乎一定命中（任何包含字母 k 的答案都会触发）。
结果：`must_not_mention: ["proof-of-work"]` 永远被错误"触发"，无辜答案被判 fail。

## 修复

```ts
needle.split(/\s*(?:或|\/|\bor\b|\|)\s*/i)
```

只对 ASCII 单词 `or` 加 `\bor\b`：
- `or` 单独成词时仍切（"A or B"）
- `or` 在 `proof-of-work` 中前后都是 `o`/`r` 同类字符，`\b` 不命中，不切

中文"或"、`/`、`|` 不需要词边界，因为它们本就是非字母符号。

## 教训

正则的"成本看起来便宜"是错觉：
- `or` 这种 ASCII 单词作分隔符必须加边界
- 修这种 bug 时单测要覆盖"分隔符出现在英文单词内部"的反例
- 当前 scorer.test.ts 已加用例防回归
