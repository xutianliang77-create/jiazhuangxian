# ASK-062 参考答案

## 解读

`errorClass: 'auth'` + `errorMessage: '401 Unauthorized'` → OpenAI 拒绝认证。chain.ts 立刻 fallback 到 anthropic 并成功。

## 根因（按概率高→低）

| 假设 | 验证 |
|---|---|
| API key 过期 / revoke | OpenAI dashboard 看 key 状态；`curl https://api.openai.com/v1/models -H "Authorization: Bearer $OPENAI_API_KEY"` |
| API key 输错（多空格 / 引号） | `echo "$OPENAI_API_KEY" \| head -c 10` 看前缀是否 `sk-` |
| 余额用尽 → key 自动 revoke | dashboard usage tab 看本月余额 |
| 配置文件指错 env var | `grep apiKeyEnvVar ~/.codeclaw/providers.json` 看名字 |
| .env 没被 source | `printenv OPENAI_API_KEY` 直接看 shell 环境 |

## 修复

1. dashboard rotate / 充值
2. 更新 `.env` 或 `providers.json`
3. 重启 codeclaw 让 provider 重新 init

## 不要做

- **关 fallback**：fallback 是 chain 的设计目的，不该关
- **删审计**：审计行为不影响认证，删了反而丢历史
- **改 errorClass**：把 auth 强转 transient 会导致每次都重试同 key，浪费配额

## 验证修复

跑一次小请求，看 chain attempts 第一条 `ok: true`：
```bash
codeclaw "say hi" 2>&1 | grep attempts
```
