# ASK-072 参考答案

## 等价性

两种调用**完全等价**：

```
/lint                ↔ /skills use lint-fix
```

loader 启动时遍历每个 skill manifest 的 `commands[]` 数组，把每条 `name` 注册到 slash registry，handler 等价于 `useSkill(skillName)`。

## 流程

1. `manifest.yaml` 里：
   ```yaml
   name: lint-fix
   commands:
     - name: /lint
       description: 触发 lint-fix skill
   ```
2. loader 启动时：`slashRegistry.add('/lint', () => useSkill('lint-fix'))`
3. 用户输入 `/lint` → registry hit → 执行同 `/skills use lint-fix`

## 设计意图

- 短命令：常用 skill 不必每次打全名
- 一致性：手动写 `/skills use lint-fix` 也能用
- 隔离：每个用户/项目可不同别名

## 限制（loader 强制）

- `name` 必须以 `/` 开头，正则 `/^\/[a-z][a-z0-9_-]{0,30}$/i`
- 不能与 builtin slash 重名（`/mode`、`/cost`、`/forget` 等）—— loader 会 reject
- 同一进程内多个 skill 注册同名 → 后者 reject

不是 shell 子进程 / exec 等其他机制。
