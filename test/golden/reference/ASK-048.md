# ASK-048 参考答案

## 参考实现

```python
from contextlib import contextmanager
import time

@contextmanager
def timed_section(name: str):
    """记录进入到退出的耗时；正常 / 异常路径都打印。"""
    start = time.perf_counter()
    try:
        yield
    finally:
        elapsed_ms = (time.perf_counter() - start) * 1000
        print(f"section {name} took {elapsed_ms:.1f} ms")


# === Demo ===
if __name__ == "__main__":
    with timed_section("normal"):
        time.sleep(0.05)

    try:
        with timed_section("raises"):
            time.sleep(0.02)
            raise ValueError("boom")
    except ValueError as e:
        print(f"caught: {e}")
```

## 关键点

| 要点 | 解释 |
|---|---|
| `@contextmanager` 装饰器 | 比手写 `class` 实现 `__enter__/__exit__` 简洁；推荐写法 |
| `yield` 一次 | 这是 generator-based context manager，必须 yield 恰好一次 |
| `try/finally` | 保证异常时也走清理逻辑（打印耗时） |
| **不在 finally 吞异常** | finally 块不显式 except / re-raise，异常自动向上传播 |
| `time.perf_counter()` | 单调高精度（>= μs），适合测短耗时；`time.time()` 在 NTP 调时会跳，不够稳 |

## 反例

```python
# 反例 1：手写 class（题目要求 contextmanager 装饰器）
class TimedSection:
    def __init__(self, name): self.name = name
    def __enter__(self): self.start = time.time(); return self
    def __exit__(self, *exc): print(...)

# 反例 2：用 time.time() 测 ms 级耗时（可能负值）
start = time.time()  # 改成 perf_counter 更稳

# 反例 3：finally 里 try/except 吞掉异常
finally:
    try:
        print(...)
    except:  # 错：会掩盖 print 的潜在错误
        pass
```

## 评分要点

- 必须用 `@contextmanager` + `yield`（不能手写 class）
- 必须 try/finally
- 必须 perf_counter / monotonic（不强制，但 time.time 也可接受）
- 异常透传（不吞）

## 常见扣分

- 手写 `__enter__/__exit__` class：-3
- 没 try/finally，异常时不打印：-2
- 在 finally 里 swallow 原始异常：-3
