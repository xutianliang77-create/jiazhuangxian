# ASK-025 参考答案

## 含义

`tokenFile` 默认是 `~/.codeclaw/wechat-ibot/default.json`，里面是 iLink 微信登录的 access token + cookie 序列化。

`mode=0644` = `rw-r--r--`：
- owner 可读写
- **group 也可读**
- **其他用户也可读**

`expected 0600` = `rw-------`：仅 owner 可读写。

## 为什么 600 必须

token 等价于"登录凭据"。本机其他用户（多用户 Linux / 容器共享 host）能 cat 到这文件就能：
- 直接拿 token 调 iLink API 冒充你发消息
- 提取 cookie 后 web 端登录

合规层面：任何 secret 文件最低 600。许多审计扫描工具（如 lynis、bench tools）会报 mode > 600 的 secret 为高危。

## 修复

```bash
chmod 600 ~/.codeclaw/wechat-ibot/default.json
codeclaw doctor    # 重新验证
```

或者重启 codeclaw —— 启动时 `ensureIlinkWechatTokenFile`（`src/channels/wechat/token.ts`）会再走一遍迁移路径，自动 chmod 600。

## 防止再发生

- 如果文件是手动创建 / cp 进来的，就要小心 cp 默认沿用源 mode
- 注意 umask：`umask 022` 会导致新建文件默认 644；secret 目录里 `umask 077` 更稳

## 不该做

- `rm` 文件：要重走 iLink QR 登录流程，麻烦
- "换数据库"：与本问题完全无关
