# ASK-058 参考答案

## 必须包含的 pattern

`REQUIRED_GITIGNORE_PATTERNS = ["node_modules", ".codeclaw", "doc", ".env"]`

## 风险排序（高→低）

| # | 漏掉 | 风险 |
|---|---|---|
| 1 | `.env` | API key / DB 密码 / 加密 secret 直接进 git history。一旦 push 公网，所有泄露密钥要立刻 rotate；GitHub 自动扫描 + 通知，但已 fork 的副本拿不回来 |
| 2 | `.codeclaw` | sessions.jsonl 含**用户全部对话内容**（含贴入的代码 / 敏感信息 / provider 配置缓存）；data.db / audit.db 也在里面 |
| 3 | `node_modules` | 仓库体积膨胀（GB 级），clone 慢；native 二进制（如 better-sqlite3）不跨平台；偶尔含构建期下载的 token |
| 4 | `doc` | 用户明示本地专用文档不入库（含项目设计 / runbook） |

## nightly 行为

- 只**检查并警告**，不主动改 `.gitignore`（避免误判覆盖用户特殊配置）
- 退出码非 0 时 CI 可拒绝合入

## 不应包含的 pattern

- `.git/`：git 自身目录，不需要在 .gitignore（git 不会提交自己）
- `README.md`：要入库
