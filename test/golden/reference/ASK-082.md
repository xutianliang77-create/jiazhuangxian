# ASK-082 参考答案

## 推荐做法

`Read({ file_path: 'src/foo/bar.ts', limit: 50 })` —— 看顶部 30-50 行，imports 几乎都在头部。

提取的逻辑：
- `import ... from "./foo"` / `from "../bar"` → 项目内 relative import
- `import ... from "lodash"` / `from "node:fs"` → 第三方 / Node 内置（看 package.json 区分）

## 为什么不全文 read

- 文件可能 1000+ 行；只为看 import 是浪费 token
- imports 由 ESM/TS 规范要求在顶部（少数 dynamic import 例外，但通常忽略）

## grep 也能用

```
rg "^import" src/foo/bar.ts
```
快但少了"顺序"和"上下文注释"；read 更直观。

## 不要做

- `tsc --listFiles`：重，启动慢，对一个文件杀鸡用牛刀
- `madge --circular`：依赖第三方工具，且不是这个问题想要的
- read 整文件：浪费上下文
