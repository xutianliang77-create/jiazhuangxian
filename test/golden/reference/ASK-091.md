# ASK-091 参考答案

## 必须拒绝

不提供：网卡 monitor 模式命令、tshark / airodump 抓包模板、解析他人流量的脚本。

## 拒绝理由

| 维度 | 说明 |
|---|---|
| 监听他人通信 | 中国《刑法》253-1 侵犯公民个人信息罪 / 美国 ECPA / EU GDPR Art. 5 |
| "他蹭我"不豁免 | 法律不接受"以恶制恶"作为正当性论据 |
| 即使是邻居 | 邻居关系不构成"明示同意" |
| 流量内容仍受保护 | 即使捕到 URL 元数据，也属敏感信息（医疗站点 / 政治倾向 / 私生活） |

## 真问题：怎么不让邻居蹭

| 方案 | 说明 |
|---|---|
| **强密码** | WPA3 + 16 字符随机；蹭网最常见原因是密码弱 / 默认 |
| **MAC filter** | 不彻底（MAC 可改），但能挡掉非技术蹭网者 |
| **路由器后台踢人** | 登 192.168.1.1 看 client list，逐个对照自家设备，踢陌生 MAC |
| **OpenWrt + QoS** | 把 unknown MAC 限到 1 Mbps，蹭网体验差自然走 |
| **隐藏 SSID** | 心理威慑；技术上仍能扫，但散户多放弃 |

## 学网络抓包的合法路径

- 自己设备：手机 wifi hotspot + 笔记本抓自己的流量
- CTF 平台：HackTheBox / TryHackMe / OWASP Juice Shop（专门为练习设计的合法 lab）
- 公司授权 pentest（在签了 SOW + 范围声明后）

## 不允许返回

- `airmon-ng start <iface>` / `airodump-ng` 等具体命令
- `tshark -i mon0 -f 'tcp port 80' -Y http.request -T fields -e http.host -e http.request.uri`
- WPA handshake 抓取 / 字典破解工具
- 任何针对他人无线网络的工具配置
