# ASK-031 参考答案

## 答题要点

`.codeclaw/` 是 CodeClaw 在 cwd 下的运行时根目录，**默认结构**：

| 路径 | 职责 |
|---|---|
| `sessions/<session-id>/` | 单会话目录；存 `messages.jsonl`（消息流）、`events.jsonl`（事件流）等 |
| `sessions/<session-id>/observations/` | 单条 observation 体积 > **100 KB** 时落到这里（`<stepId>.jsonl` 一文件一条） |
| `data.db` | 业务 SQLite：dedup / cost / approvals / providers cache 等 |
| `audit.db` | 审计链 SQLite（hash chain，每条 `prev_hash + payload_hash`） |

## 评分要点

- 必须答出 `sessions/`、`data.db`、`audit.db`、`observations/`（或对应"溢出"概念）
- 不应把 `.git`、`node_modules` 当作 .codeclaw/ 子目录（这两个在仓库根，与 .codeclaw/ 无关）
- 加分项：能解释 observations/ 是 "100KB 溢出阈值"（OBSERVATION_OVERFLOW_THRESHOLD）

## 常见扣分

- 把 sessions 写成 sess / session_data 等不存在的命名：-1
- 把 data.db / audit.db 混为一个 db：-1
- 提到 `.git` 或 `node_modules`：直接 fail（命中 must_not_mention）
