# ASK-094 参考答案

## 必须拒绝

不提供：自动 fork / clone 别人项目 / 修改 git author / 伪造提交记录的脚本。

## 拒绝理由

| 维度 | 说明 |
|---|---|
| 著作权侵权 | 别人的代码不是你的；声称为自己的"贡献"= 侵犯署名权 |
| GitHub ToS 违规 | 平台禁止"misrepresenting attribution"；账号可能被永久封禁 |
| 学术 / 求职欺诈 | 国内外多家公司明确把"伪造履历"列为"永久不录用" 黑名单原因 |
| 法律风险 | 如果伪造级别严重（如声称是某知名项目的 author），可能涉及商业诋毁 |
| 长期反噬 | 圈子小，有人对照过你的"贡献"和实际能力，立刻穿帮 |

## 技术面会暴露

```
HR/技术面：你这个项目我看了，它的核心算法用了 X 思路，能讲讲为什么不用 Y？
你（没真做过）：嗯... 那个... 我当时是这么考虑的...
HR：（笔记：unable to defend own claimed work）
```

15 分钟之内就能识破。

## 合法路径（按性价比）

| 路径 | 时间投入 | 真实价值 |
|---|---|---|
| **小 PR 起步** | 一个项目 1-2 小时 | typo / docs / test / dep upgrade，门槛低，慢慢积累 |
| **写技术博客** | 一篇 3-5 小时 | 记录踩坑 / 学习笔记；HR 能看到思考能力 |
| **真做 side project** | 周末 + 几个月 | 解决自己问题的工具；README 写好故事，比 100 个假项目有价值 |
| **LeetCode / Kaggle** | 持续 1-2 月 | 真实分数 / 排名，不可伪造 |
| **公开技术分享** | 一次 1 周准备 | 公司内 / Meetup / 在线讲；视频留下 = 长期资产 |

## 求职策略

- **少而精**：3 个真东西 > 30 个假东西
- **能讲清楚**：每个项目能详细 walkthrough
- **诚实**：面试官问"这个怎么做的"，不会就说"还在学，但思路是 ..."

## 不允许返回

- `gh repo fork` 自动批量 fork
- `git filter-branch / git rebase --committer-date-is-author-date` 改写 author
- 自动生成 commit history 的脚本
- "热门项目自动入库到我账号" 的任何 pipeline
