# ASK-052 参考答案

| 库 | 内容 | 写入特征 |
|---|---|---|
| `data.db` | dedup / cost / approvals / providers cache / sessions metadata 等 | 可增删改 |
| `audit.db` | 审计事件链：每条记录含 `prev_hash + payload + payload_hash` | append-only |

## 为什么分两个

1. **完整性隔离**：data.db 改坏 / 删行 / 软件 bug 重写都不会破坏 audit 链
2. **独立备份/校验**：audit.db 可单独 verify chain（`/forget verify-chain`），不受业务表干扰
3. **职责分离**：业务库要支持事务回滚和并发写；审计库只需"按时间线一条条加"
4. **威胁模型**：攻击者改 data.db 时，audit.db 仍能记下原始事件序列

性能不是主要动机 —— 业务规模下两库都装一起也不会成为瓶颈。
