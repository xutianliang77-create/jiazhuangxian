# ASK-018 参考答案

## 参考代码

```ts
import { monotonicFactory } from "ulid";

const ulid = monotonicFactory();
const ids = Array.from({ length: 5 }, () => ulid());

for (let i = 1; i < ids.length; i++) {
  if (!(ids[i] > ids[i - 1])) {
    throw new Error(`monotonic broken at ${i}: ${ids[i - 1]} >= ${ids[i]}`);
  }
}

console.log("ok", ids);
```

## 评分要点

- 用 `monotonicFactory()` 拿到 generator（**不**是直接 `import { ulid }`）
- 字符串 `>` 比较即可（ULID 设计上字典序==时间序）
- 断言失败抛 Error（题目要求）
- ESM `import` 而非 `require`
- 不要混用 UUID v4 / v7

## 为什么不用 localeCompare

ULID 字符集是 Crockford Base32（数字+大写字母排除易混淆字符），ASCII 序与字典序一致，普通 `>` 就够。`localeCompare` 引入 locale 不确定性，反而出问题。

## 常见扣分

- 调了 `ulid()` 而非工厂返回的实例：单次调用是单调的，但跨调用不保证
- 用 `setTimeout(... , 1)` 隔毫秒：避开了 monotonic 真正解决的问题（同毫秒）
- 改成"跨毫秒生成"：题意是验证同毫秒下也单调
