# ASK-037 参考答案

## 子命令

### `codeclaw skill list`
- 无参数
- 输出：builtin skills + user 已装 skills，含 name / version / type 列
- builtin 包含：`review` / `explain` / `patch` / `data_insight`（4 个）

### `codeclaw skill install <local-path>`
- `<local-path>`：本地目录，必须含 `manifest.yaml`
- 流程：
  1. 读 manifest，跑 `validateManifest`（schema + commands 校验）
  2. 拒绝与 builtin 同名的 skill（防覆盖）
  3. `cpSync(local-path, ~/.codeclaw/skills/<name>/)` 全量拷贝
- 不会跑 `npm install` / 不下载远端依赖（这是本地 cp）

### `codeclaw skill remove <name>`
- `<name>`：user skill 名（不带路径）
- **builtin 不可 remove**：会拒绝并给提示
- 删除 `~/.codeclaw/skills/<name>/` 整个目录

## 安装位置

`~/.codeclaw/skills/<skill-name>/` —— 与 builtin（在 dist 内）独立。

## 不存在的命令

- `skill update`：没有；要更新就 remove + install
- `skill publish`：没有
- 没有 `npm install` 路径，没有远端 registry

## 评分要点

- 三个子命令名 + 各自参数必须答对
- ~/.codeclaw/skills/ 路径必答
- builtin 不可 remove 必答
- 不应虚构 update / publish

## 常见扣分

- 把 install 说成 npm install：-2
- 漏说 builtin 保护：-1
