# ASK-039 参考答案

## 实现

CodeClaw Web channel 基于 **Node 内置 `node:http`**（`http.createServer`）—— 见 `src/channels/web/server.ts:22` 的 import 与 line 231 的 `http.createServer((req, res) => ...)`。

注释明确写："基于 Node 内置 http；零额外依赖（不引入 express / koa / ws）。"

## 为什么不用 Express / Koa / Fastify

| 理由 | 说明 |
|---|---|
| **零依赖** | CodeClaw 是 CLI 工具，依赖树越小：分发包小、安全审计面小、CVE 风险低、安装快 |
| **路由极简** | 只有 ~8 条路由（sessions/messages/stream/cost/providers）+ 静态文件，express 的中间件/插件生态用不上 |
| **SSE 控制精细** | 直接 `res.write('data: ...\n\n')`、`res.flushHeaders()`、心跳定时器；框架的 stream 抽象反而多一层 |
| **启动快** | 没有 framework init / 中间件链，CLI 子进程启动 < 100ms |
| **历史 CVE** | express / koa 都有历史 CVE（path-to-regexp ReDoS 等）；Node 内置 http 维护节奏更稳 |

## 评分要点

- 必须答 "Node 内置 http" / `node:http` / `http.createServer`
- 至少给出 2 个理由
