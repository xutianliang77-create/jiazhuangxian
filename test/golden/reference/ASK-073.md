# ASK-073 参考答案

## 业务清理

| 命令 | 范围 |
|---|---|
| `/forget --all` | 跨表清空 dedup / cost / approvals / sessions metadata |
| `/forget --session <sid>` | 单会话清理（保留其他会话） |
| `/forget --kind dedup` | 只清单表（如已实现） |

## 审计验证

| 命令 | 作用 |
|---|---|
| `/forget verify-chain` | 走完整 hash 链校验，找出第一个断点（如有） |
| `/forget verify-chain --verbose` | 详细打印每条 event 的 prev_hash / payload_hash 比对 |

## 局部撤销

| 命令 | 作用 |
|---|---|
| `/forget approval <id>` | 撤回单条已批准的审批（写 tombstone） |

## 不存在

- `/forget --hard`：不允许"硬删 audit" 的子命令
- `destroy db`：拒绝直接销毁 db 的语法

设计原则：业务可清，审计**永远** append-only。
