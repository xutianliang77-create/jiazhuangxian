# ASK-019 参考答案

## 规则本体

`SlashRegistry.match` 取 prompt 第一个 token（按空白切），与命令名做**精确**比较。
等价的字符串判定是：

```
prompt === "/foo"  ||  prompt 以 "/foo " 开头（"/foo" + 空白）
```

即"完整 token + 空白边界"。

## 解决的 bug 类型

纯 `startsWith` 时：
- 注册 `/foo` 后，用户输入 `/foobar baz` 会被 `/foo` 抢截
- handler 拿到的 args 会是 `bar baz`（错位）
- `/foobar` 自己永远跑不到，因为前缀冲突总让 `/foo` 先命中

更广义的 prefix 串味：`/log` vs `/login`、`/list` vs `/listen` 等都会出事。

## 不是大小写问题

注册 + 匹配都已经做了 toLowerCase，与 case 无关。

## 用 regex 也能做

`/^\/foo(\s|$)/` 也能实现"前缀 + 边界"，但比一个 `===` + `startsWith(" ")` 复杂、更易写错。当前实现胜在简单。
