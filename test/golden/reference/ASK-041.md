# ASK-041 参考答案

## 推荐工具与参数

```
Read({ file_path: "/var/log/app.log", offset: 8000, limit: 1000 })
```

- 工具：`read`（按行偏移读取，结构化返回）
- 参数：`offset=8000`（起始行）、`limit=1000`（行数）
- 返回：第 8000-8999 行内容（带行号前缀）

## 为什么不读全文

1. **token 上下文**：1 MB 日志若 30 字节/行 ≈ 35000 行；放进 prompt 会消耗上 10万 token，挤掉真正的对话上下文
2. **单工具响应大小限制**：read 工具单次返回有上限（默认 ~2000 行），全文读会被截断
3. **冗余**：用户只想看一段，多读 = 浪费 LLM 推理算力

## 次优方案

```bash
sed -n '8000,9000p' /var/log/app.log
# 或
awk 'NR>=8000 && NR<=9000' /var/log/app.log
```

缺点：
- 经过 shell escape，风险面更大
- 返回不带行号（除非 `sed -n '8000,9000{=;p}'` 复杂写法）
- 没有 read 工具的结构化输出

## 反例（错误做法）

```bash
cat /var/log/app.log     # 全文进上下文，立刻撑满
head -9000 /var/log/app.log | tail -1000   # 双工具链，多一次进程
```

## 评分要点

- 必须答 read + offset/limit 参数
- 必须解释为什么不全读（token / 上下文）
- 加分项：给出 sed/awk 次优方案

## 常见扣分

- 推荐 `cat /var/log/app.log`：直接 fail
- 没说明 offset/limit 具体值：-2
