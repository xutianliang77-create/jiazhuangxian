# ASK-016 参考答案

## 两个工具的能力边界

### `/symbol` + `/references`（语义路径）
- 走 LSP（默认 `fallback-regex-index`，安装 `multilspy` 后切真实 LSP）
- 能区分**同名变量在不同 scope**、识别**真正的调用**而非字符串字面量
- **限制**：每种语言要单独起 LSP 服务（TS: typescript-language-server / Py: pyright），多语言混搭仓库覆盖不齐

### `/grep`（文本路径）
- 纯 regex 文本搜索，跨语言一刀切，对"调用方都用同名 API"的场景够用
- **限制**：会命中注释、字符串字面量、文档；可能误报"看起来在调用其实没"

## 实战建议

```
1. /grep <oldApiName>            # 拉候选清单（cheap，跨语言）
2. /references <oldApiName>      # 主语言里做语义确认（去掉误报）
3. 把"两边都命中"的当真，"只 grep 命中"的人工二次审
```

## 反例

- "用 ls 即可"——ls 列文件名，不查内容
- "直接读所有文件"——大仓 O(N×M) 不现实
