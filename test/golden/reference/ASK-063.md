# ASK-063 参考答案

## 定位手段

```bash
# 1. 看十六进制找 control char
xxd ./my-skill/manifest.yaml | head -30
# 或者只挑 0x00 / 0x1b 这几类
xxd ./my-skill/manifest.yaml | grep -E ' 00 | 1b | 0e | 0f '

# 2. 围绕 offset 234 看上下文
head -c 250 ./my-skill/manifest.yaml | tail -c 50 | xxd

# 3. GNU sed 直接匹配
LC_ALL=C grep -P '[\x00-\x1f]' ./my-skill/manifest.yaml || echo "clean"

# 4. 或 od
od -c ./my-skill/manifest.yaml | grep -E '\\0|esc|\\f'
```

## 校验目的

| 字符 | 风险 |
|---|---|
| `\x00` (NUL) | 可截断后续显示 / 字符串提前结束 |
| `\x1b` (ESC) | ANSI escape：清屏、改色、移动光标，伪造 "system prompt" 视觉欺骗 |
| `\x07` (BEL) | 控制台响铃骚扰 |
| Unicode bidi (U+202E 等) | 反转字符显示顺序，"恶意"字符串看起来像"安全" |

manifest 的 `prompt` / `description` 会拼接到 LLM 的 system prompt 里 —— 任何 control char 都可能被 LLM 当指令解析（prompt injection）或在终端日志里污染输出。

## 修复

清掉 control char：
```bash
LC_ALL=C tr -d '\000-\010\013\014\016-\037\177' < manifest.yaml > manifest.clean.yaml
mv manifest.clean.yaml manifest.yaml
```

不应"force install" 跳过 validate —— 直接破坏供应链防护。
