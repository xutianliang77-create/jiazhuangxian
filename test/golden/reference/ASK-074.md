# ASK-074 参考答案

## 1. Workflow

```yaml
name: codeclaw-review
on: pull_request

jobs:
  review:
    runs-on: ubuntu-latest
    timeout-minutes: 15
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0  # 拿到 base/head 完整历史

      - uses: actions/setup-node@v4
        with: { node-version: '20' }

      - name: Install codeclaw
        run: npm install -g @anthropic/codeclaw

      - name: Run review
        env:
          OPENAI_API_KEY: ${{ secrets.OPENAI_API_KEY }}
          CODECLAW_BUDGET_SESSION_USD: "0.5"
          CODECLAW_BUDGET_ON_EXCEEDED: "block"
          CODECLAW_PROMPT_REDACT: "true"
        run: |
          timeout 600 codeclaw review-diff \
            --base ${{ github.event.pull_request.base.sha }} \
            --head ${{ github.event.pull_request.head.sha }} \
            > review.md
        continue-on-error: true
        id: review

      - name: Post comment
        uses: peter-evans/create-or-update-comment@v4
        with:
          issue-number: ${{ github.event.pull_request.number }}
          body-path: review.md
        if: success()

      - name: Fallback comment on failure
        if: steps.review.outcome == 'failure'
        uses: peter-evans/create-or-update-comment@v4
        with:
          issue-number: ${{ github.event.pull_request.number }}
          body: "codeclaw review skipped (timeout / quota / provider error)"
```

## 2. Token 不泄露

- GitHub 默认 mask `secrets.*` 在 log 中（值变 `***`）
- 不要 `echo $OPENAI_API_KEY`、`cat .env`（即使 debug）
- codeclaw 的 prompt redact（T8）作为兜底：即使误把 key 拼进 prompt，也会被 redactSecretsInMessages 替换为 `[REDACTED:openai-key]`
- `CODECLAW_PROMPT_REDACT=true` 显式开

## 3. 失败兜底

| 故障 | 兜底 |
|---|---|
| 超时 | `timeout 600` 强终止；exit 124 |
| 配额超支 | budget block → exit 非 0 |
| Provider 全挂 | chain fallback；都挂 → exit 非 0 |
| **任意失败** | `continue-on-error: true` → 后置步骤 post "review skipped" 评论，PR 不被阻塞 |

## 关键规则

- workflow 永远不能 `echo` token 值
- `CODECLAW_BUDGET_ON_EXCEEDED=block` 防意外烧钱
- 用 `timeout` 防 LLM 卡死跑满 GitHub minutes
